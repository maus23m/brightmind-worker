# Coverage Taxonomy Demo — Year 7: Expressions & Equations

**Source of granularity:** KS3 statutory programme of study (Algebra) + NCETM KS3 non-statutory guidance, Year 7 framework. The *list* of sub-strands is curriculum fact (free to use). Every question below is **original**, written fresh to model the pattern — nothing lifted from any source.

This is exactly what the Curriculum Agent would distil per topic: the **width** (sub-strands) and **depth** (cognitive bands), with one worked example per cell. Use these as golden-set seeds (B-list) and CR-027 few-shot examples.

---

## Width — sub-strands
1. Algebraic notation & conventions
2. Forming expressions from context
3. Substitution
4. Simplifying (collecting like terms)
5. Expanding a single bracket
6. Solving linear equations

**Design note (the judgement that matters):** "one-step vs two-step equations" is **depth, not width**. It's the same skill at increasing demand, so it lives *inside* sub-strand 6 as the depth progression — not as separate sub-strands. Getting this call right is the whole point of a human taxonomy sign-off: split too finely and the grid explodes; lump too coarsely and you miss real gaps.

## Depth bands
- **Recall** — knows the convention/fact directly
- **Procedure** — executes a standard method, single step
- **Application** — applies in context or combines two steps
- **Reasoning** — multi-step, work backwards, justify, or find the error

## The grid (6 × 4 = 24 cells)

### 1. Algebraic notation & conventions
| Depth | Question | Answer |
|---|---|---|
| Recall | What does the expression `5y` mean? | 5 multiplied by y (5 lots of y) |
| Procedure | Write "a number `n` multiplied by itself" using index notation. | n² |
| Application | A pencil costs `p` pence. Write an expression for the cost of 4 pencils. | 4p |
| Reasoning | Jordan writes "double a number, then add 3" as `a3 + 2`. Explain what is wrong and write it correctly. | Two errors: `a3` is not valid notation for "double a" (that is `2a`), and a number should be written before the letter. Correct: `2a + 3`. |

### 2. Forming expressions from context
| Depth | Question | Answer |
|---|---|---|
| Recall | Which expression means "3 more than `x`": `x + 3` or `3x`? | x + 3 |
| Procedure | Write an expression for "7 less than a number `m`". | m − 7 |
| Application | A taxi charges £3 plus £2 per mile. Write an expression for the cost of a journey of `m` miles. | 3 + 2m |
| Reasoning | Sam is `x` years old. His sister is 3 years younger; his dad is twice Sam's age. Write and simplify an expression for their combined age. | x + (x − 3) + 2x = 4x − 3 |

### 3. Substitution
| Depth | Question | Answer |
|---|---|---|
| Recall | If `a = 5`, what is the value of `a + 2`? | 7 |
| Procedure | Find the value of `3n − 4` when `n = 6`. | 14 |
| Application | The cost of hiring a bike is `C = 2h + 5` pounds, where `h` is hours. Find the cost for 4 hours. | £13 |
| Reasoning | When `x = 4`, two students evaluate `2x²`. One gets 32, the other 64. Which is correct, and what mistake did the other make? | 32 is correct (2 × 4² = 2 × 16). The other squared `2x` as a whole (8² = 64) instead of squaring only x. |

### 4. Simplifying (collecting like terms)
| Depth | Question | Answer |
|---|---|---|
| Recall | Are `4a` and `3a` like terms? | Yes — same variable |
| Procedure | Simplify `5x + 2x − x`. | 6x |
| Application | Simplify `4a + 3b + 2a − b`. | 6a + 2b |
| Reasoning | A student simplifies `5x + 3` to `8x`. Explain the mistake. | `5x` and `3` are not like terms (one has an x, one is a constant), so they cannot be combined. It stays `5x + 3`. |

### 5. Expanding a single bracket
| Depth | Question | Answer |
|---|---|---|
| Recall | When expanding `3(x + 2)`, what is each term inside the bracket multiplied by? | 3 |
| Procedure | Expand `4(x + 3)`. | 4x + 12 |
| Application | Expand `5(2y − 1)`. | 10y − 5 |
| Reasoning | Expand and simplify `3(x + 4) + 2(x − 1)`. | 3x + 12 + 2x − 2 = 5x + 10 |

### 6. Solving linear equations *(depth = step complexity)*
| Depth | Question | Answer |
|---|---|---|
| Recall | Check whether `x = 3` is the solution of `x + 4 = 7`. | Yes — 3 + 4 = 7 |
| Procedure (one-step) | Solve `x + 5 = 12`. | x = 7 |
| Application (two-step) | Solve `3x + 4 = 19`. | x = 5 |
| Reasoning (form & verify) | I think of a number, multiply it by 4, then subtract 5. The result is 23. Form an equation, solve it, and check your answer. | 4n − 5 = 23 → 4n = 28 → n = 7. Check: 4 × 7 − 5 = 23 ✓ |

## How this feeds the build
- **CR-022** — this grid *is* the coverage skeleton for one topic. A child's matrix lights up cell-by-cell; untested cells are visible to the parent.
- **Golden set (B-list)** — these 24 verified pairs are seed entries, once a curriculum specialist signs off.
- **CR-027** — verified cells become few-shot seeds, so the generator produces new questions *of the right kind* per cell.
- **CR-026** — note which cells are deterministically checkable (substitution, simplifying, expanding, solving = yes; notation/explanation reasoning = needs judge/human). That split tells you where the deterministic floor reaches.

**One topic, one year = 6 sub-strands, 24 cells.** Multiply across the curriculum and you see both the scale of the asset and why the launch should narrow to a few topics first.

CLEAN-ANSWER RULE — applies to "mean", "median" and "percent_of":
The verifier rejects a question whose computed answer matches no option. Division-based ops produce messy decimals unless you pick the numbers deliberately. Before choosing inputs, check the result is a whole number or a decimal with at most one place.

- "median": with an even count the median is the average of the two middle values — make those two sum to an even number (36 and 42 → 39 ✓; 35 and 42 → 38.5, only use if 38.5 is an option).
- "percent_of": (part × 100) must be divisible by whole ([24, 80] → 30% ✓; [3, 7] → 42.857…% ✗).

COORDINATE GRID QUESTIONS — SPECIAL RULE:
For any question that asks the child to READ coordinates from a diagram (e.g. "What are the coordinates of point P?", "Which point has coordinates (3, -2)?"), the label list MUST NOT include the coordinate values of the points being identified. Label each plotted point with its letter name ONLY (e.g. "Point P", "Point A", "Point B"). The child reads the coordinates from the grid axes — that is the whole point of the question. Putting "P(3, 2)" or "(3, 2)" next to the point in the label list makes the answer immediately visible and the question worthless.

The trick: you CAN describe WHERE to place the point in the prose paragraph (the drawing agent needs that to position it correctly) — but the label list must only contain the letter name. Like this:

"Draw a coordinate grid with x-axis from -4 to 4 and y-axis from -4 to 4. Plot a single orange dot at the grid intersection three units right and two units up from the origin. Label it 'P'.
Label the diagram with below information (and nothing more):
1. X-axis labelled with integer values -4 to 4
2. Y-axis labelled with integer values -4 to 4
3. Point P (letter name only — no coordinates)
4. Axis labels 'x' and 'y'
5. Texts should be away from drawings (proportionally away but near)
6. No overlaps between two objects unless its legit intersection"

The drawing agent reads "three units right and two units up" and places the dot at (3, 2). The child then reads (3, 2) off the axes. Nobody put "(3, 2)" in a label.

GRID-AREA / COUNT QUESTIONS — SPECIAL RULE:
For any question where the child must COUNT discrete items shown in a diagram (filled grid squares, tiles, dots in an array), the diagramPrompt MUST name every single item by its exact position. The number of items named in the prose MUST equal inputs.length in the compute block. The drawing agent has NO knowledge of the compute block — it draws only what the prose describes. If you name 4 squares in the prose but inputs has 9 entries, the diagram will show 4 and the answer will say 9. That is the bug.

Rule: count what you draw, draw what you count.

For grid-area questions, describe each filled square using "row R col C" notation (row 1 = top row, column 1 = leftmost column). List ALL of them explicitly in the prose before the label whitelist. Do NOT use shape names alone ("an L-shape", "a T-shape") — a shape name gives the drawing agent artistic freedom to draw a different number of squares than inputs.length.
